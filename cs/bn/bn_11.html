01 Transport Layer

전송계층은 신뢰성 있는 데이터 전송과 효율적인 네트워크 통신을 담당하는 계층으로, 다음 네 가지 핵심 역할을 수행한다:

  1. 세그먼트화(Segmentation)
  2. 전송 제어(Transmission Control)
  3. 흐름 제어(Flow Control)
  4. 다중화(Multiplexing)

1. 세그먼트화(Segmentation)

세그먼트화는 상위 계층에서 전송되는 데이터를 작은 단위로 나누고, 수신 측에서 이를 재조립하는 과정이다.

  • 송신 측: 큰 데이터 스트림을 효율적 전송을 위해 작은 세그먼트로 분할
  • 수신 측: 세그먼트를 원래 데이터 스트림으로 재조립하여 상위 계층에 전달

목적: 전송 효율성 증가, 오류 복구 용이, 신뢰성 있는 데이터 전달 보장

2. 전송 제어(Transmission Control)

전송 제어는 송신자와 수신자 간의 신뢰성 있는 데이터 전송을 보장한다.

  • 오류 제어(Error Control): 전송 중 발생한 데이터 손상이나 손실을 검출하고, 재전송 요청을 통해 데이터 무결성을 확보
  • 혼잡 제어(Congestion Control): 네트워크 과부하 시 전송 속도를 조절하여 혼잡을 방지하고 성능 저하 최소화

3. 흐름 제어(Flow Control)

흐름 제어는 데이터 전송 속도를 조절하여 수신 측 버퍼 오버플로를 방지하고 네트워크 혼잡을 줄인다.

  • Stop-and-Wait 방식: 송신 측에서 한 패킷 전송 후 수신 측 확인 응답을 기다린 후 다음 패킷 전송
  • 슬라이딩 윈도우(Sliding Window) 방식: 여러 패킷을 연속 전송하고 수신 측의 확인 응답을 통해 전송을 관리

4. 다중화(Multiplexing)

다중화는 여러 애플리케이션 계층 데이터를 하나의 전송 채널로 통합하는 과정이다. 각 세그먼트에는 식별용 포트 번호를 포함하여 수신 측에서 올바른 애플리케이션으로 전달된다.

  • 포트(Port): IP 주소 내 논리적 통신 지점, 애플리케이션 식별 및 데이터 분배 역할
  • 소켓(Socket): 실제 데이터 송수신 통로 제공. 서버와 클라이언트 간 세션 연결 관리

5. 전송 계층 프로토콜

5.1 TCP(Transmission Control Protocol)

  • 연결 지향형 프로토콜
  • 신뢰성 있는 데이터 전달 및 순서 유지, 흐름 제어 및 혼잡 제어 지원
  • 3-way handshake를 통한 연결 설정
  • 사용 사례: 웹, 이메일, 파일 전송 등
  • 헤더 구조: 복잡하지만 신뢰성 확보에 필수적

5.2 UDP(User Datagram Protocol)

  • 비연결형 프로토콜
  • 빠른 전송 속도, 최소 헤더 구조, 신뢰성 보장 없음
  • 사용 사례: 스트리밍, 실시간 게임
  • 헤더 구조: 송수신 포트, 길이, 체크섬 등 최소 정보 포함

6. TCP 연결 설정 과정

TCP 연결은 3-way handshake를 통해 이루어진다:

  1. SYN: 클라이언트 → 서버, 연결 요청
  2. SYN-ACK: 서버 → 클라이언트, 연결 수락 및 응답
  3. ACK: 클라이언트 → 서버, 연결 확정

각 단계에서 시퀀스 번호(Seq)와 확인 응답 번호(Ack)를 사용하여 신뢰성을 유지하며, 데이터 흐름과 재전송 관리를 지원한다.
