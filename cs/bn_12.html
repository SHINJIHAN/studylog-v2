OSI 표현 계층과 응용 계층

OSI 모델의 표현 계층(Presentation Layer)은 데이터의 변환, 암호화, 압축 등 상위 애플리케이션 간 데이터 호환성을 보장하는 역할을 수행한다. 이를 통해 하드웨어 및 운영체제 독립성을 유지하고, 다양한 표현 방식을 표준화할 수 있다.

1. 응용 계층(Application Layer)

응용 계층은 최종 사용자와 직접 상호작용하며, 데이터의 구조화, 처리, 전송을 담당한다.

1.1 HTML과 마크업 언어

  • HTML: 웹 콘텐츠 구조화, 확장성 및 호환성 제공
  • HTML5: 멀티미디어 통합, Canvas API, 반응형 디자인, 로컬 스토리지, WebSocket 지원으로 현대 웹의 표준화 구현

1.2 CGI(Common Gateway Interface)

  • 플랫폼 독립적이며 언어 제약이 없음
  • 요청 시마다 프로세스 생성으로 성능 저하 발생 가능
  • 유지보수 어려움과 보안 취약점 존재

1.3 XML(eXtensible Markup Language)

  • 구조화된 데이터 저장 및 전송에 최적화
  • 사용자 정의 태그 지원, 가독성 우수, 플랫폼 독립성 확보

웹 시스템 구조와 아키텍처

웹 시스템의 주요 목표는 데이터 처리 효율, 확장성 확보, 보안 및 안정성 유지이다.

2. 핵심 구성 요소

  • 클라이언트(Client): 사용자 요청 생성 및 화면 표시
  • 웹 서버(Web Server): HTTP 요청 처리, 정적 콘텐츠 제공
  • 애플리케이션 서버(App Server): 비즈니스 로직 처리
  • 데이터베이스(DB): 데이터 저장 및 관리

2.1 확장 및 최적화 구성 요소

  • 로드 밸런서(Load Balancer): 서버 부하 분산
  • 캐시 서버(Cache Server): 반복 데이터 빠른 접근 제공
  • CDN(Content Delivery Network): 글로벌 데이터 전송 최적화
  • 보안 시스템(Security Layer): 접근 제어 및 위협 방어

2.2 아키텍처 진화

단일 계층 → 2계층 → 3계층 → 마이크로서비스 아키텍처로 발전, 모듈화 및 확장성 강화

웹 요청 및 응답 흐름

3. DNS(Domain Name System)

  • 필요성: 기억하기 어려운 IP 주소를 도메인 이름으로 변환, 유연한 변경, 효율적 운영
  • 구조: 최상위 루트 → TLD → 세컨드 레벨 도메인 → 서브 도메인 → 호스트, 계층적 트리 구조
  • 쿼리 방식: Recursive, Iterative
  • 활용 사례: 웹 브라우징, 이메일, CDN, 부하 분산, 보안

이메일과 FTP 프로토콜

4. 이메일 시스템

  • 흐름: 메일 작성 → 송신 → 라우팅 → 저장 → 수신
  • 프로토콜: SMTP(송신), POP3/IMAP(수신)
  • MIME(Multipurpose Internet Mail Extensions): 다양한 언어, 이미지, 동영상 지원

5. POP3 vs IMAP

  • POP3: 서버에서 메일 다운로드 후 로컬 저장, 단순 구조
  • IMAP: 서버에서 메일 동기화 및 관리, 멀티 디바이스 지원

6. FTP(File Transfer Protocol)

  • 동작 모드: 액티브 모드, 패시브 모드
  • 보안 강화 프로토콜: FTPS, SFTP, FTP over SSL/TLS 등 안전한 파일 전송 지원
