01 Subnet Mask

IP 주소에서 네트워크와 호스트 영역을 명확히 구분하기 위한 핵심적인 비트 마스크이다. 이를 통해 네트워크 식별, 호스트 구분, 라우팅 효율화가 가능하며, IPv4 주소 체계에서 필수적인 구성 요소로 작동한다.

1. 네트워크 분할

  • IPv4 주소는 32비트로 구성

  • ‘1’: 네트워크 영역

  • ‘0’: 호스트 영역

  • 서브넷팅(Subnetting):

  • 기존 네트워크를 보다 작은 서브넷으로 분할하여 네트워크 관리 효율과 보안을 향상시키고, 라우팅 최적화를 달성한다.

  • 네트워크 비트 수 증가 → 서브넷 수 증가, 호스트 수 감소.

  • 슈퍼넷팅(Supernetting):

  • 다수의 작은 네트워크를 하나의 큰 네트워크로 결합하여

  • CIDR(Classless Inter-Domain Routing) 구현 및 라우팅 테이블 축소 효과를 달성한다.

2. 계층적 라우팅

Hierarchical Routing 라우팅 테이블의 항목 수를 최소화하여 라우터 부하를 줄이고, 네트워크 관리 효율성을 높인다. 이를 통해 네트워크 범위 식별, 라우팅 영역 구분, 상위 네트워크 및 외부 네트워크와의 효율적인 데이터 전달이 가능하다.

장점: 주소 활용 효율화, 보안 강화, 트래픽 최적화, 계층적 설계. 단점: 설계 복잡성 증가, 오류 가능성, 관리 부담 증가.

3. 연계 요소

게이트웨이, DNS, 라우팅 테이블과 밀접히 연관되어 네트워크 통합 관리와 안정적 운영을 지원한다.

02 네트워크 주소 변환 프로토콜

IP와 MAC 주소 간 변환 및 네트워크 장치 초기화, IP 갱신 등 다양한 상황에서 필수적이다.

1. ARP

Address Resolution Protocol

  • IPv4 환경에서 IP 주소를 MAC 주소로 변환.
  • 송신자 확인 → 브로드캐스트 요청 → 유니캐스트 응답 → 캐시 저장.
  • 특징: 브로드캐스트-유니캐스트 혼합 통신, 일정 기간 IP-MAC 매핑 캐시, 편리하지만 보안 취약점 존재.

2. RARP, BOOTP, DHCP

  • RARP: MAC 주소를 기반으로 IP 주소 요청.
  • BOOTP: IP 주소가 없는 장치에 초기 IP 할당.
  • DHCP: BOOTP 기능을 확장하여 IP, 서브넷 마스크, 게이트웨이, DNS 등 자동 제공.
  • 동작 순서: Discover → Offer → Request → Acknowledge.
  • 일반 가정용 및 기업 환경에서 널리 사용.

03 기타 네트워크 프로토콜

1. ICMP

Internet Control Message Protocol

  • IP 네트워크에서 오류 보고와 제어 메시지 전달용 보조 프로토콜.
  • IP 자체에는 오류 보고 기능이 없으므로, ICMP를 통해 안정적 네트워크 동작을 지원.
  • 주요 메시지: 오류 보고, 상태 진단, 경로 추적 등.

2. IGMP

Internet Group Management Protocol

  • IPv4에서 멀티캐스트 그룹 관리. IPv6에서는 MLD 사용.
  • 필요성: 멀티캐스트 트래픽 효율적 관리, 실시간 서비스 지원.
  • 동작: 가입 요청 → 라우터 정보 유지 → 주기적 멤버 확인 → 탈퇴 알림.
  • 버전별 특징: v1 기본 가입 기능, v2 빠른 탈퇴 지원, v3 SSM(Source-Specific Multicast) 지원.
  • 적용 사례: 유튜브 라이브 스트리밍, 금융 거래 서비스.

3. 라우팅 프로토콜

  • 내부 라우팅 프로토콜: OSPF(Link State 기반), 빠른 수렴, 계층적 네트워크 구조에 적합.
  • 외부 라우팅 프로토콜: BGP(자율 시스템 간 경로 선택), 인터넷 백본 및 대규모 네트워크에서 표준으로 사용.
